        1. Accessibility - Målet med accessibility är att göra det enklare för personer med funktionsnedsättningar att tillgodogöra sig innehåll
        på webben. Det handlar om såväl design som funktionalitet, och även om utvecklingen gått långsamt har många länder idag
        infört formella riktlinjer och till och med lagar avseende hur innehåll ska struktureras för att "alla" ska ha möjlighet att
        använda webbinnehåll på ett jämlikt sätt. I ljuset av detta blir det tydligt varför utvecklaren bör sträva efter att skapa lättillgängliga
        webbsidor.

        2. Landmarks - Indelningar av webbinnehåll som underlättar för skärmläsare att återge innehållet på ett mer
        lättbripligt sätt för personer med nedsatt syn. Exempel kan vara semantiska element såsom <header> och <footer>
        
        3. Semantic elements - Semantiska element är ännu ett sätt att - under paraplybegreppet accessibility - göra webbinnehållet tillgängligt för fler.
        Mer konkret handlar det om att märka upp html-dokumenten med taggar som bättre representerar den slags information som de innehåller. 
        Ett exempel kan vara att använda <section>, <main>, <aside>, <header> och <footer> istället för att genomgående använda <div>:ar.
        
        4. Role attribute - Role-attributet är ytterligare ett verktyg som utvecklaren kan dra nytta av för att göra innehåll mer lättillgängligt
        för alla användargrupper. Det finns situationer då det inte går att använda de typiska hjälpmedelsmetoderna (t.ex. de som beskrivits ovan).
        Det kan till exempel handla om begränsningar i vissa verktyg eller ramverk som inte tillåter att vi använder ett visst semantiskt element.
        För att kringgå den här typen av problem och ändå kunna tillgodose behoven hos alla slutanvändare, inklusive personer med funktionhinder, 
        är det vanligt att använda attribut såsom role, och även name och value. Om vi t.ex. antar att vi av någon anledning inte kan använda elementet <button>, så kan vi istället använda oss av role="button", som skärmläsaren har lätt att hantera.
